# Настройка Proxy API

Если вы используете Proxy API (прокси-сервис для доступа к OpenAI), вам нужно настроить файл `.env` следующим образом:

## Шаг 1: Создайте файл .env

Используйте один из способов:
- **Автоматически:** `python create_env.py` (выберите вариант 2)
- **Вручную:** создайте файл `.env` в корне проекта

## Шаг 2: Добавьте Proxy API данные

Откройте файл `.env` и добавьте следующие строки:

```env
PROXY_API_KEY=ваш_proxy_api_ключ
PROXY_API_URL=https://api.ваш-прокси-сервис.com/v1
```

### Примеры:

#### Пример 1: Если используете распространенный прокси-сервис
```env
PROXY_API_KEY=sk-abc123xyz789
PROXY_API_URL=https://api.openai-proxy.com/v1
```

#### Пример 2: Если используете собственный прокси
```env
PROXY_API_KEY=your_proxy_key_here
PROXY_API_URL=https://your-proxy-server.com/v1
```

## Важные моменты:

1. **НЕ добавляйте** `OPENAI_API_KEY` если используете Proxy API
2. Убедитесь, что **оба** параметра заполнены:
   - `PROXY_API_KEY` - ваш ключ от Proxy API
   - `PROXY_API_URL` - URL вашего Proxy API сервиса
3. URL должен заканчиваться на `/v1` (стандартный endpoint OpenAI API)

## Как работает выбор API:

Система автоматически определит, какой API использовать:

- Если указаны **PROXY_API_KEY** и **PROXY_API_URL** → будет использоваться Proxy API
- Если указан только **OPENAI_API_KEY** → будет использоваться прямой OpenAI API
- Если указаны оба → приоритет у Proxy API

## Проверка настройки:

После настройки `.env` файла, запустите:

```bash
python run.py
```

Если все настроено правильно, сервер запустится без ошибок.

## Где взять Proxy API данные:

1. **API Key:** Обычно предоставляется провайдером Proxy API сервиса
2. **API URL:** URL endpoint вашего Proxy API сервиса (обычно в документации провайдера)

---

**Примечание:** Если у вас есть вопросы по конкретному Proxy API сервису, обратитесь к документации вашего провайдера.
